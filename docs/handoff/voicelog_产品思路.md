# 语迹 VoiceLog · 产品思路梳理

> Slogan：**时间是你最大的资产**
>
> 这份文档回应你抛出的那些想法。我的核心工作不是照单全收，而是帮你分清——哪些是真正的杀手锏、哪些会反过来毁掉产品。把它们放进一个有取舍的产品框架里。

---

## 一、先确立产品的"一句话灵魂"

你的 Slogan「时间是你最大的资产」其实已经把产品方向点透了。它意味着：

VoiceLog 不是又一个日历，也不是又一个待办清单。它是一个**帮你把时间当资产来经营的工具**——记录时间花在哪、复盘是否花得值、并主动帮你把时间投到更高回报的地方。

这条灵魂会帮你判断每个功能该不该做：**凡是能让用户更清醒地经营时间的，做；凡是只是炫技、或反而帮用户逃避选择的，砍。**

---

## 二、你的想法分级（这是这份文档的核心）

我把你提的所有想法按"价值 × 可行性"分成四档。

### 🟢 第一档：立刻做，低成本高回报

**1. 撤销按键（Undo）**
- 价值：基础但极其重要。语音误识别、手滑删错日程是高频场景，没有撤销会让用户不敢用。
- 实现：极简。前端存一个操作栈，撤销就回退上一步。半天工作量。
- 判断：**必做，且应该早做。** 它是"信任感"的地基。

**2. 允许上传文件、图片，由大模型解析其中的日程**
- 价值：高。这是一个真实的高频痛点——课程表截图、会议邀请邮件、活动海报、行程单，里面全是日程，手动录入很烦。
- 实现：中等。图片/PDF → 多模态大模型 → 提取结构化日程 → 进你现有的日程流。技术成熟，按需调用，成本可控。
- 判断：**强烈建议做，这是一个能让人"哇"的差异化功能。** 它把"录入"这个最大的摩擦点几乎抹平。

### 🟡 第二档：做，但要克制、要设计

**3. 智能自动修改日程 / 自动排班给选项**
- 价值：中高。但**这里有个陷阱**：市面上 Akiflow、Motion、Reclaim、Focuster 全在拼"自动 time-blocking"，这是红海，你拼不过也没必要拼。
- 我的建议：**别做"自动重排"，做"建议式排班"。** 区别在于——你不替用户做决定，而是当用户的日程冲突或太满时，**给出 2-3 个选项让用户选**。这既轻量、又契合你"经营时间"的灵魂（用户始终是决策者），还避开了和大厂正面竞争。
- 判断：**做轻量版（给选项），不做重量版（全自动重排）。**

**4. 帮助 prioritize 目标**
- 价值：高，且非常契合 Slogan。但**"优先级"是个深坑**——做浅了是摆设，做深了是另一个产品（OKR 工具）。
- 我的建议：用最轻的方式切入。比如让用户给日程打一个简单的"重要度"标记，复盘时告诉他「你这周 80% 时间花在了低重要度的事上」。**让优先级服务于复盘，而不是单独做一套目标管理体系。**
- 判断：**做，但绑定在复盘里做，别单独立项。**

### 🟠 第三档：是你的"灵魂功能"，但要小心翼翼地做

**5. 允许重叠日程，但提醒用户 multitasking 对前额叶的损伤**
- 价值：**这是你所有想法里最有"产品人格"的一个。** 它把一个普通的日程工具，变成一个有立场、关心用户的工具。它直接呼应了 Slogan——多任务是在浪费你最宝贵的资产。
- 风险：**说教感。** 如果每次重叠都弹一段"前额叶损伤"的科普，用户会烦到卸载。
- 我的建议：做，但要**极度克制和优雅**。
  - 允许重叠（不强行阻止，尊重用户）。
  - 重叠时用一个**轻量、非说教**的提示，比如一句话 + 一个可关闭的小标记，而不是大段科普。
  - 把"深度工作 vs 碎片多任务"的洞察**沉淀到复盘里**——周复盘告诉用户「你这周有 6 小时是在多任务中度过的」，比当场弹窗有效得多，也不惹人烦。
- 判断：**做，这是你的差异化人格。但用"复盘里点出"代替"当场说教"。**

### 🔴 第四档：危险想法，我建议你重新考虑

**6. 接入"直接打开 App / 强制终止娱乐 App"的权限**
> （到学习时刻，程序立即终止你所有 entertainment 页面，自动打开对应 App）

我必须诚实地拦你一下，这个想法有三个层面的问题：

- **技术上几乎不可行（尤其 iOS）**：iOS 的沙盒机制**根本不允许**一个 app 去关闭或控制另一个 app。这是苹果的核心安全设计，没有任何正规 API 能做到"强制终止别人的娱乐 app"。安卓也极受限。你想做也做不出来。
  - （唯一沾边的是 iOS 的"屏幕使用时间 / 专注模式"和快捷指令，但那是**用户自己设置的系统级功能**，不是你的 app 能主动控制的，体验也完全不同。）
- **产品哲学上是反的**：你的 Slogan 是"时间是你最大的资产"——资产是用来**经营**的，不是被**没收**的。一个会强行关掉你 app、夺走你手机控制权的工具，本质上是在剥夺用户的自主权。用户会感到被监管、被惩罚，而不是被赋能。这和"帮你清醒地经营时间"是相反的。
- **会摧毁信任**：一个能强行操控你手机的 app，是用户最恐惧的东西。

**我的建议：保留"意图"，换掉"手段"。**
你真正想解决的是「到了该专注的时候，帮我抵抗娱乐的诱惑」。这个意图是好的。但实现方式应该是**赋能而非强制**：
- 到了专注时刻，**主动提醒** + 一键帮用户开启 iOS 自带的"专注模式"（通过快捷指令，用户授权过的）。
- 用**正向激励**：完成一段专注，给用户看"你今天守住了 2 小时深度时间"。
- 让用户**自己预先设定**规则（"工作日 9-11 点提醒我别刷视频"），到点温和提醒——而不是 app 替他强行执行。

判断：**原始想法（强制控制）砍掉，它既做不到也不该做。保留其意图，用"提醒 + 引导 + 正向激励"实现。**

## 三、把这些想法整合进产品的优先级路线

基于上面的分级，给你一条务实的演进路线：

**阶段 1 · 把地基打牢（现在的网页版 + 这些）**
- 撤销按键（信任地基）
- 文件/图片上传解析日程（抹平录入摩擦，制造惊喜）
- 复盘里绑定"重要度"，点出时间是否花得值

**阶段 2 · 注入产品人格**
- 允许重叠日程 + 优雅的多任务提示（差异化人格）
- 建议式排班（冲突时给选项，不替用户决定）
- 专注时刻的"提醒 + 引导专注模式"（赋能版，非强制版）

**阶段 3 · 深化经营感（验证留存后再做）**
- 更聪明的目标优先级与复盘联动
- 跨周期的时间资产趋势（你这个季度的时间投资回报）

## 四、贯穿始终的设计原则（从你的想法里提炼出来的）

1. **赋能，不强制。** 用户永远是时间的主人，app 是顾问不是监工。（这条直接否决了"强制关 app"，也指导了"建议式排班"。）
2. **复盘是主场，说教是禁区。** 所有"对用户好"的提醒（多任务、低优先级），都尽量沉淀到复盘里温和呈现，而不是当场弹窗教育。
3. **抹平摩擦优先于增加功能。** 撤销、文件解析这类"减少麻烦"的功能，比炫酷的自动化更能留住用户。
4. **每个功能都问一句：它让用户更清醒地经营时间了吗？** 是，就做；只是炫技或帮用户逃避选择，就砍。

---

## 五、一句话总结我的判断

你的想法里，**文件解析**和**多任务复盘提示**是真正的宝藏，应该重点做；**撤销**是必须补的地基；**自动排班**和**优先级**要做轻量版、别陷进红海；而**强制控制手机**那个想法，技术上做不到、哲学上也和你的 Slogan 相反，建议彻底换成"提醒 + 引导"的赋能版本。

把握住一条主线：**VoiceLog 帮用户像经营资产一样经营时间——它给用户看清时间的去向、提供更优的选择，但永远把决策权留给用户。**
