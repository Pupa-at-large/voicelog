# 语迹 VoiceLog · 产品需求文档（PRD v1）

> **Slogan：时间是你最大的资产**
>
> 产品一句话：VoiceLog 不是管理你的"时间"，是管理你的"成长"。日程是输入，复盘是引擎，真正的产品是——让你看见自己在一点点变好的那条上升曲线。像经营游戏角色一样经营自己，升级、复盘、向上。
>
> 本文档合并了竞品研究、功能探讨结论、以及等级系统设计稿，作为 Phase 1 开发前的对齐基准。

---

## 一、核心定位（已对齐）

**VoiceLog = 语音驱动的"时间成长系统"**，而非排班器。

- 它**不**和 Motion / Akiflow / Reclaim 拼"自动排班"——那是红海，且需要算法和资本壁垒。
- 它站在一个巨头因战略包袱不会来的位置：**为重视复盘、重视自主、说中文的人，做一个帮他们经营时间、看见自己成长的工具。**
- 差异化三支柱：**复盘（看清时间去向）+ 成长（等级与上升曲线）+ 本地优先（数据属于你，永不锁定）**。

### 壁垒的清醒认知
单看功能没有传统技术壁垒（语音是 API、解析是规则、复盘是统计）。真正的壁垒在：
1. **定位壁垒**：巨头掉不了头来做这个垂直定位。
2. **价值观壁垒**：赋能而非强制、把决策权留给用户——这是有人格的产品，抄了就不是 Motion。
3. **数据壁垒**（长期最硬）：复盘有记忆，越用越懂你，迁移成本越来越高。
4. **本地化 + 离线**（短期可守）：中文口语精细打磨 + 真离线，巨头懒得做。

> 但最终生死 99% 取决于"用户会不会每天用"。壁垒是做大后的事；现在先用零成本网页版验证真实留存。

## 二、竞品研究结论（精简版）

### 主流 task manager 的启示
| 产品 | 强在 | 弱在 | 给我们的启示 |
|---|---|---|---|
| Todoist | 快速捕获、自然语言、简单可靠 | AI 落后、好功能锁付费墙 | "简单可靠"比花哨更留人 |
| Notion | 极致灵活、替代多工具 | 学习曲线陡、要自己搭 | 别学万能，要开箱即用 |
| Motion | 全自动排班 | 贵、AI"咄咄逼人"、不离线 | 它的缺点就是我们的机会 |
| Akiflow | 键盘流、聚合工具、**有 rituals 复盘** | 贵、不离线 | 复盘是它的边角料，却是我们的核心 |
| Reclaim | 习惯保护、团队排会 | 集成少、控制弱 | "习惯保护"思路可借鉴 |

### 三个关键发现
1. **power user 离开的头号原因不是缺功能**，而是：更新破坏工作流、定价突变、同步失败。→ **速度、透明、尊重用户投入 > 功能堆砌**。
2. **本地不锁定是市场甜点**：Obsidian 的本地 Markdown 在竞品差评里被反复当优点提及。→ **我们天然本地优先 + Markdown 导出，正好踩中**。
3. **Typeless 的精髓值得借鉴**：它把"AI 懂你"可视化成可追的进度（个性化 14.3%）、隐私叙事降低恐惧、越用越懂你。→ **直接启发了我们的等级报告页**。

### Gamification 的数据与红线
- 有用：多邻国靠它把流失从 47% 砍到 28%。
- **但 streak 是陷阱**：断 2-3 天的用户只有 0.9% 回归。裸 streak = 心理陷阱。
- **惩罚机制赶走新用户**（Habitica 角色会"死"）。
- **红线（已定）**：只做正向微奖励、等级只升不降、**绝不做惩罚和裸 streak**。这既健康又抗作弊，呼应 "late better than never"。

| 专注拦截（Screen Time）| 远期原生 | **原生**（需 Mac+特批）| 3+ |

### 关键设计原则（贯穿所有功能）
1. **赋能，不强制。** 用户永远是时间的主人，app 是顾问不是监工。
2. **复盘是主场，说教是禁区。** "对用户好"的提醒（多任务、低优先级）沉淀到复盘里温和呈现，不当场弹窗。
3. **抹平摩擦 > 增加功能。** 撤销、找回、文件解析这类减摩擦功能最留人。
4. **奖励"做了"而非只奖励"做完"。** late better than never。等级只升不降。
5. **每个功能都问：它让用户更清醒地经营时间、更看见自己变好了吗？**

### 批量语音指令 / 多意图解析（重点新增）
这是"丝滑体验"的核心来源，对标 Typeless 那种"叽里咕噜说一通，AI 全办了"。

**场景**：用户一次说一大段话，里面混合了多种意图，AI 一次性全部执行：
- 编排**多条新日程**（"明天三点开会、五点健身、晚上八点和朋友吃饭"）
- **补录之前漏记的**（"哦对，昨天下午那个评审会其实开了"）
- 给**已完成的打勾**（"今天上午的方案我写完了"）
- 顺手**记下备忘**（"开会记得带合同"）

**交互设计**：解析后**先展示一个可编辑的"待执行清单"**（新增 X 条、完成 X 条、补录 X 条），用户一眼扫过、可改可删，确认后批量执行。**永不静默执行**——呼应"需确认"原则。

**技术现实**：
- 单靠本地规则引擎做不好。"一句一条"规则能扛；但"一段话混合多意图"需要 LLM 拆解。
- 因此定位 **Phase 2、接 API**。Phase 1 先把"一句一条"做扎实。
- 兜底：LLM 失败时退回逐句规则解析，永不崩。

### 被否决/降级的想法
- **强制关闭娱乐 App**：iOS 沙盒根本无此 API，"同意"也做不到。→ 砍掉。
- **经授权的拦截**（Screen Time API 的 shield）：可做，但需 Mac + 苹果特批资格 + 应对 API 不稳定，是重功能。→ 降级到 Phase 3+ 原生版。
- **全自动排班/重排**：Motion 红海且招骂。→ 只做"给选项 + 需确认"的建议式轻量版。

## 四、等级系统设计（已细化）

### 机制
- **等级 = 时间管理者的成长阶段**，由累计 XP 决定。
- **XP 来源**：记录日程、完成日程、做复盘——**只增不减**。
- **每级有称号**（如 LV.4「稳步前行的人」），给身份感。
- **升级有庆祝时刻**（升级动画 = 你要的"升级打怪"奖励）。
- **不用 streak / 连续天数**，改用「**累计天数**」等只增不减的指标，规避断签清零的挫败。

### 报告页结构（最终稿，配色已定）
设计稿见对话中的可视化（深墨绿等级面板 + 暖金等级色 + 低饱和数据色）。从上到下：
1. **等级面板**：等级徽章 + 称号 + 经验条（"740 XP · 距 LV.5 还差 60"）。— 核心，成长的"养成感"在这里。
2. **三项数据**：本周记录 / 完成率 / 累计天数。
3. **时间去向**：分类条形图（深度工作/会议沟通/学习成长/生活健康）。
4. **本周洞察**：一句正向、非说教的话，奖励进步。
5. **隐私承诺**：数据本地存储、随时导出、等级只升不降。

### 视觉风格
- 高级、克制：深墨绿 + 暖金，低饱和数据色，靠留白和层次显质感，不靠炫光。
- 延续现有 demo 的深青墨色 UI 语言，保持一致。

## 五、Phase 规划

### Phase 1（地基 + 验证留存）— 几乎全本地，零成本
- 已有：日程解析、增删改提醒、基础复盘、导出。
- 新增：**撤销、删除找回、打钩微奖励动画、可关分类、等级系统雏形（XP+等级显示）**。
- 接入：**云端语音识别 API**（唯一需要的外部依赖，需一个 serverless 中转藏 key）。
- 目标：**验证"你和身边几个人会不会每天用"**。这是最重要的里程碑。

### Phase 2（注入差异化与人格）
- 文件/图片解析日程、丝滑复盘、成长报告页、长程任务里程碑、建议式排班、多任务提示、AI 调整需确认。
- 目标：把"复盘 + 成长"的护城河立起来。

### Phase 3（深化 + 原生）
- 自适应个性化、专注拦截（原生版，需 Mac + 特批资格）。
- 目标：做成真正的"时间成长系统"。

## 六、技术架构（在线网页方案）

```
① 语音识别 → 在线 API（约 ¥5/月级别）｜兜底：浏览器 Web Speech API（免费）
② 日程解析 → 本地规则引擎（零成本、离线、已有）
③ 复盘+等级 → 本地统计为主，深度洞察按需调 API
④ 提醒推送 → PWA 弱推送验证；认真做时换 Telegram Bot
```
- 前端：网页 / PWA（手机加到主屏幕，复用现有 UI）。
- 中转后端：Serverless（Vercel 免费额度），仅用于藏 API key。
- 存储：本地（SQLite/本地文件）+ Markdown 镜像导出。
- 部署：Vercel / Netlify / GitHub Pages，零成本拿到网址。
- 月成本预估：**1 美元以内**，验证阶段近乎免费。
