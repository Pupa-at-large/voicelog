## 1. 这是什么项目

**语迹 VoiceLog** — 一个语音驱动的「时间成长系统」。
- Slogan：**时间是你最大的资产**。
- 一句话：不是管理"时间"，是管理"成长"。日程是输入，复盘是引擎，真正的产品是让用户看见自己一点点变好的上升曲线。像经营游戏角色一样经营自己——记录、复盘、升级。
- **不和 Motion/Akiflow 拼自动排班**（红海），而是做"复盘 + 成长 + 本地优先"的差异化定位。

---

## 2. 当前进度（做到哪了）

**已完成 Phase 1 的可运行原型**：`voicelog_phase1.html`（纯前端单文件，浏览器打开即用）。

已实现并测试通过的功能：
- 语音/文字输入 → 中文口语解析成结构化日程（标题/时间/地点/提醒），**纯本地规则引擎**
- 日程增删改、定时提醒、备注
- 复盘（日/周/月/季/年）：时间去向分类条形图 + 完成率 + 文字洞察 + 导出 MD/TXT
- **等级系统**：XP（记录+5/完成+10/复盘+15，只升不降）、LV.1~10 称号、经验进度条、"成长"标签页、升级庆祝动画
- **打钩微奖励**：勾选弹跳动画 + XP 飘字
- **撤销**：顶部撤销按钮 + 操作后 Toast 内"撤销"按钮；已完成日程再点一次可取消完成
- **删除找回**：软删除进回收站，可找回/彻底删除
- **可关 AI 分类**：自动给日程打类别标签，设置里可开关

技术现状：
- **纯前端、无后端、无依赖**，所有逻辑（解析/复盘/等级）都是真实 JS 代码，跑在浏览器里。
- 数据存在内存（刷新重置）—— 这是原型，**还没做持久化**。
- 语音用浏览器免费的 Web Speech API，**还没接云端语音 API**。

## 4. 关键设计原则（务必遵守，这些是反复讨论后定下的）

1. **赋能，不强制。** 用户永远是时间的主人，app 是顾问不是监工。
2. **复盘是主场，说教是禁区。** "对用户好"的提醒（多任务危害、低优先级）沉淀到复盘里温和呈现，绝不当场弹窗说教。
3. **抹平摩擦 > 增加功能。** 撤销、找回、容错这类功能要让用户在犯错的那一刻、那个位置就能用到。
4. **gamification 红线**：只做正向微奖励、**等级/XP 只升不降**，**绝不做惩罚机制、绝不做会清零的连续打卡（streak）**。奖励"做了"而非只奖励"做完"（late better than never）。
5. **本地优先**：数据属于用户、可导出、永不锁定。这是 vs Motion 的差异化武器。

## 5. 接下来做什么（建议的下一步）

### 即时可做（工程化原型）
当前 `voicelog_phase1.html` 是单文件原型。要变成真正能持续开发的项目，建议：
- [ ] 把单文件 HTML 拆成正经前端项目结构（建议 Vite + 原生 JS 或轻框架，**保持轻量**）
- [ ] 加数据持久化（先 localStorage 或 IndexedDB，验证后再上后端/iCloud）
- [ ] 改造成 PWA（manifest + service worker），让 iPhone 能"添加到主屏幕"
- [ ] 部署到 Vercel/Netlify（免费），拿到可访问网址

### Phase 2（PRD 里已规划）
- [ ] 接云端语音 API（推荐 `gpt-4o-mini-transcribe`，约 $0.003/分钟）——需一个 serverless 中转藏 API key
- [ ] **批量语音指令 / 多意图解析**：一段话同时建多条日程 + 补录 + 打勾 + 备注（需 LLM 拆解，先出可编辑的"待执行清单"再批量执行，永不静默执行）
- [ ] 文件/图片上传 → 多模态大模型解析日程
- [ ] 丝滑复盘（读用户备注 + 结束时问体验）
- [ ] 长程任务里程碑 + 进度条
- [ ] 建议式排班（给选项/需确认，**不做全自动重排**）
- [ ] 成长报告页（Typeless 式"越来越懂你"的可视化，设计稿见对话/PRD）

### 待用户拍板的开放问题
- 等级称号文字（现有一套初稿在 `voicelog_phase1.html` 的 `LEVELS` 数组，用户要改）
- 语音 API 选型（Whisper vs gpt-4o-mini-transcribe）
- 第一版是否先纯本地免费语音验证留存，再接 API

## 6. 重要的现实约束（别踩坑）

- **iOS 原生 app 必须用 Mac + Xcode**，Windows 做不了最后的签名上架。当前走"网页/PWA"路线正是为了绕开这个，先零成本验证。
- **"强制关闭娱乐 App"做不到**——iOS 沙盒无此 API。"经用户授权的拦截"可用 Screen Time API 做，但需 Mac + 苹果特批资格，是 Phase 3+ 的重功能。
- **提醒在纯 PWA 上不可靠**（iOS Web Push 限制）。认真做时考虑 Telegram Bot 推送。
